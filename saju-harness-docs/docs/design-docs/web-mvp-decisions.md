# Web MVP Decisions

## Decision

초기 제품은 네이티브 앱이 아니라 반응형 웹 서비스로 구현한다.

## Rationale

- 배포와 수정 속도가 빠르다.
- 사용자가 링크로 바로 진입할 수 있다.
- 사주 리포트와 궁합 결과를 공유하기 쉽다.
- 앱스토어 심사와 네이티브 배포 과정을 MVP에서 피할 수 있다.
- SEO와 콘텐츠 확장 가능성이 있다.

## Product Impact

웹 MVP에서는 다음 경험을 우선한다.

- 모바일 웹에서 빠른 온보딩
- 결과 리포트 URL 공유
- 친구/연인에게 링크 전달
- 궁합 CTA로 자연스러운 재진입
- 추후 PWA 또는 네이티브 앱 확장 가능성 유지

## Out of Scope

- App Store 배포
- Play Store 배포
- 네이티브 푸시 알림
- 네이티브 인앱 결제
- iOS/Android 전용 UI 컴포넌트

## Future Options

웹 MVP 이후 다음 방향을 검토할 수 있다.

1. PWA 전환
2. 카카오톡 공유 최적화
3. 네이티브 앱 래핑
4. iOS/Android 앱 별도 개발
