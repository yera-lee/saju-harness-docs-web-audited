# PRODUCT_SENSE.md

## Product Positioning

이 서비스는 사주를 기반으로 사용자의 연애 성향, 관계 패턴, 궁합, 자기이해를 돕는 분석형 웹 서비스이다.

이 웹 서비스는 단순한 오늘의 운세 서비스이 아니다.  
또한 사용자를 불안하게 만드는 예언형 점술 서비스도 아니다.

핵심 방향은 다음과 같다.

- 사주를 자기이해 도구로 사용한다.
- 연애와 관계를 주요 진입 포인트로 삼는다.
- 궁합 기능을 자연스러운 확장 기능으로 제공한다.
- 분석 결과는 단정이 아니라 해석과 조언의 형태로 제공한다.

## Core User Question

사용자가 이 웹 서비스에 들어오는 가장 중요한 질문은 다음과 같다.

> 나는 왜 비슷한 연애를 반복할까?

이 질문에서 출발해 다음 질문으로 확장한다.

- 나는 어떤 사람에게 끌리는가?
- 나는 관계에서 어떤 패턴을 반복하는가?
- 나와 잘 맞는 사람은 어떤 사람인가?
- 그 사람과 나는 잘 맞을까?
- 지금 내 관계 흐름은 어떤 상태인가?

## Target Users

### Primary Users

- 20~40대 사용자
- 사주, 운세, MBTI, 타로, 심리테스트에 관심이 있는 사람
- 연애, 궁합, 인간관계에 관심이 많은 사람
- 복잡한 명리학 용어보다 쉬운 해석을 원하는 사람
- 자신의 반복되는 관계 패턴을 이해하고 싶은 사람

### Secondary Users

- 가볍게 오늘의 운세를 확인하고 싶은 사람
- 친구나 연인과 궁합을 보고 싶은 사람
- 사주는 궁금하지만 전문적인 풀이에는 진입장벽을 느끼는 사람
- 커리어, 인간관계, 재물운으로 확장된 분석을 원하는 사람

## Product Principles

### 1. 불안을 팔지 않는다

이 웹 서비스는 공포, 재앙, 실패, 이별, 사고, 질병을 단정적으로 예언하지 않는다.

잘못된 표현:
- 올해는 반드시 이별합니다.
- 당신은 결혼운이 없습니다.
- 이 사람과 만나면 불행해집니다.

좋은 표현:
- 올해는 관계에서 감정 기복이 커질 수 있어요.
- 관계의 속도를 조절하는 것이 중요합니다.
- 상대와의 차이를 이해하고 조율하는 과정이 필요합니다.

### 2. 단정하지 않고 해석한다

사주는 절대적인 결론이 아니라 사용자를 이해하기 위한 해석 프레임으로 사용한다.

권장 표현:
- 그럴 가능성이 있습니다.
- 이런 경향이 강하게 나타날 수 있습니다.
- 이 부분을 의식하면 관계가 더 편해질 수 있습니다.

### 3. 쉽고 감각적으로 설명한다

전문 용어는 최소화한다.

십신, 오행, 일간, 대운, 세운 등의 용어를 사용할 수는 있지만, 사용자가 이해할 수 있도록 쉬운 설명을 함께 제공해야 한다.

예:
- “화 기운이 강합니다”라고만 말하지 않는다.
- “표현력과 추진력이 강하지만, 감정이 빠르게 올라올 수 있습니다”처럼 풀어서 설명한다.

### 4. 연애와 관계를 첫 경험의 중심에 둔다

초기 사용자는 깊은 명리학 분석보다 자신의 연애 성향, 끌리는 사람, 반복되는 관계 패턴에 더 빠르게 반응한다.

첫 리포트는 다음 주제를 우선한다.

- 나의 연애 성향
- 내가 끌리는 사람
- 반복되는 관계 패턴
- 관계에서 조심할 점
- 나에게 맞는 관계 방식

### 5. 궁합은 자연스러운 두 번째 행동으로 설계한다

첫 리포트 이후 사용자가 자연스럽게 궁합 분석으로 이동할 수 있어야 한다.

예:
> 내 연애 패턴을 확인했다면, 이제 그 사람과의 관계 흐름도 확인해볼까요?

궁합 기능은 단순히 “좋다/나쁘다”가 아니라 다음 관점으로 제공한다.

- 끌림 포인트
- 충돌 포인트
- 대화 방식
- 관계 속도
- 장기 관계 가능성
- 조심할 감정 패턴

## MVP Focus

초기 MVP는 다음 기능에 집중한다.

1. 사용자 기본 정보 입력
2. 첫 사주 기반 연애 성향 리포트 생성
3. 반복되는 관계 패턴 분석
4. 궁합 분석으로 이어지는 CTA
5. 리포트 저장 또는 공유 기능의 기본 구조

## Non-Goals

초기 MVP에서 다음 기능은 제외한다.

- 전문 명리학자용 고급 분석 도구
- 의료, 법률, 투자 판단
- 죽음, 사고, 질병에 대한 예언
- 절대적인 결혼운/이별운 단정
- 지나치게 복잡한 만세력 UI
- 처음부터 모든 운세 카테고리를 제공하는 것

## Success Criteria

- 사용자가 3분 이내에 첫 리포트를 받을 수 있다.
- 사용자가 결과를 읽고 “내 얘기 같다”고 느낄 수 있다.
- 결과가 불안감을 조장하지 않는다.
- 첫 리포트 이후 궁합 분석으로 자연스럽게 이어진다.
- Agent가 기능을 만들 때 제품 방향을 임의로 흔들지 않는다.
